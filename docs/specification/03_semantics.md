# Семантика языка DEA

## 1. Общие сведения

1. Программа состоит из одного файла.
2. Точка входа — главная функция **`func int main()`**.
3. Язык императивный.
4. Язык статически типизированный: объявления переменных и параметров функций содержат явный тип.
5. На верхнем уровне допускаются только объявления.
   Исполняемые инструкции находятся внутри функций/процедур (включая main).

## 2. Типы данных и их особенности

В языке существуют 4 базовых типа данных:

* `int` — целые числа
* `num` — числа с плавающей точкой
* `string` — строки
* `bool` — булев тип 

**Литералы:**

* Числовой литерал без точки имеет тип `int`.
* Числовой литерал с точкой имеет тип `num`.
* Строковый литерал `"..."` имеет тип `string`.
* `true`/`false` имеют тип `bool`.

### 2.1. `int`

**Формат и диапазон:** 64-битное целое число со знаком.

**Диапазон:** -2^63 .. 2^63 - 1 

**Литерал:** Числовой литерал без точки имеет тип `int`.

**Операции:**

* унарные: `+`, `-`
* бинарные: `+`, `-`, `*`, `//`, `%`, `^`
* `/` разрешён для `int`, но результат имеет тип `num`
* разрешены встроенные функции для чисел: `abs`, `min`, `max`

* деление на ноль для `/`, `//`, `%` считается ошибкой.

### 2.2. `num`

**Формат:** `binary float 64`.

**Литерал:** Числовой литерал с точкой имеет тип `num`.

**Операции:**

* унарные: `+`, `-`
* бинарные: `+`, `-`, `*`, `/`, `^`
* * разрешены встроенные функции для чисел: `abs`, `min`, `max`

Операции `//` и `%` для `num` не поддерживаются.

* деление на ноль для `/` считается ошибкой.

### 2.3. `string`

**Кодировка:** ASCII.

**Литерал:** Строковый литерал `"..."` имеет тип `string`.

**Операции:**

* `string + string` — конкатенация, результат `string`
* другие комбинации со `string` запрещены (например, `string + int`)

**Длина и декомпозиция:**

* `len(s: string) -> int` — длина строки
* `substr(s: string, start: int, length: int) -> string` — подстрока.
* Декомпозиция на части/символы: `substr(s, i, 1)`.

**Краевые случаи и ошибки:**

* `len("") == 0`
* `substr(s, start, length)`:

  * `start < 0` или `length < 0` -> ошибка выполнения
  * `start > len(s)` -> ошибка выполнения
  * `start + length > len(s)` -> ошибка выполнения
  * `substr(s, start, 0)` возвращает пустую строку

### 2.4. `bool`

**Литералы:** `true`/`false` имеют тип `bool`.

**Операции:**

* унарный: `!`
* бинарные: `&&`, `||`
* сравнение: `==`, `!=`

### 2.5. Преобразования типов

* Явных преобразований типов нет. 
* Неявных преобразований между числами и строками нет.
* Разрешено неявное преобразование к `bool` из типов:
  * `int` и `num`: `false`, если значение равно `0`, иначе `true`
  * `string`: `false`, если строка пустая `""`, иначе `true`

## 3. Область действия и видимость

1. Существует **глобальная область видимости** для переменных/констант, объявленных на верхнем уровне, а также для объявлений функций и процедур. 
2. Существует **локальная область видимости функции/процедуры**: все переменные и константы, объявленные внутри тела функции/процедуры, видимы только в пределах этой функции/процедуры.
3. Порядок объявления определяет порядок видимости.
4. При обращении к имени из тела функции/процедуры сначала используется локальное объявление (если есть), иначе — глобальное.

## 4. Переменные и константы
   1. Объявление переменной обязательно перед использованием в данной области видимости. Использование необъявленной переменной считается ошибкой.
   2. Повторное объявление имени в одной и той же области считается ошибкой.
   3. Поддерживаются неизменяемые константы `const`.
      
      3.1. Значение константы должно быть задано в момент объявления.

      3.2. Любые попытки присваивания в const считаются ошибкой.
   4. Поддерживаются изменяемые переменные, объявляемые через явный тип, например: `int count = 5;`.
   
      4.1. Переменная может объявляться без инициализации; чтение значения такой переменной до первого присваивания считается ошибкой.

   ```dea+e
   int a;
   a = 10;

   string text = "hello!";

   const int c = 0;
   ```

## 5. Ввод/вывод
### Ввод
* Организован с помощью встроенной процедуры: `input(x)`.
* Аргумент `x` должен быть объявленной переменной (не `const`).
* Тип вводимого значения определяется типом переменной `x`.

### Вывод
* Организован с помощью встроенной процедуры: `print(x)`.
* Допускается печать значений всех базовых типов.

```dea+e
int a;
input(a);
print(a + 5);
```

## 6. Выражения

### 6.1. Выражения могут содержать:

* числовые литералы
* строковые литералы
* булевы литералы (`true`/`false`)
* операторы (бинарные и унарные)
* идентификаторы
* скобки, содержащие вложенные выражения
* вызовы пользовательских функций и встроенных функций

**Встроенные функции для чисел:**

* `abs(x)` — возвращает модуль числа
* `min(x, y, ...)` — возвращает наименьшее из переданных чисел
* `max(x, y, ...)` — возвращает наибольшее из переданных чисел

**Встроенные функции для строк:**

* `len(s)` — возвращает длину строки
* `substr(s, start, length)` — возвращает подстроку (используется и для декомпозиции: `substr(s, i, 1)`)

### 6.2. Вычисление логических выражений

Для операторов `&&` и `||` используется **короткая схема вычисления (short-circuit evaluation)**:

* `A && B`: если `A` ложно, `B` **не вычисляется**, результат — `false`.
* `A || B`: если `A` истинно, `B` **не вычисляется**, результат — `true`.

Оператор `!A` всегда вычисляет свой операнд `A`.

## 7. Виды инструкций

   1. Присваивание
   2. Отдельные инструкции объявления переменных и констант
   3. Отдельные инструкции для чтения и печати
   4. Условные инструкции
   5. Цикл while
   6. Цикл for
   7. Вызов процедуры
   8. Возврат из функции
   9. Прерывание цикла
   10. Продолжение цикла

Важно:

* **Присваивание не является выражением**, но является инструкцией. 

## 8. Разделитель инструкций

* Разделителем инструкций является символ точка с запятой `;`. 
* Разделитель не используется в конце циклов и условий.

## 9. Ветвления `if ... else`. Решение проблемы «висячего else»

Ветвление в DEA задаётся конструкцией `if` с опциональной веткой `else` и выполняется так:

* Вычисляется условие в скобках `if ( ... )`;
* Условие `if (expression)` должно иметь тип `bool` или возможность быть к нему приведенным.
* Если условие истинно — выполняется блок `{ ... }` после `if`;
* Если условие ложно и `else` есть — выполняется блок `{ ... }` после `else`, иначе управление переходит к следующей инструкции.
* Если условие ложно и `else` нет — управление переходит к следующей инструкции.

`Else` однозначно относится к ближайшему соответствующему `if` в структуре программы.

```dea+e
int age;
input(age);

if (age >= 18)
{
    print("Совершеннолетний");
}
else
{
    print("Несовершеннолетний");
}
```

## 10. Циклы

**Цикл `while`:**

* Условие цикла должно иметь тип `bool`.
* Тело выполняется, пока условие истинно.

**Цикл `for`:**

* `for (i = start to end) { ... }`
* `for (i = start downto end) { ... }`

1. Выполняется присваивание `i = start`.
2. Значение `end` вычисляется один раз при входе в цикл и фиксируется как граница.
3. Для `to`:

   * цикл продолжается, пока `i <= end`
   * после каждой итерации выполняется `i = i + 1`
4. Для `downto`:

   * цикл продолжается, пока `i >= end`
   * после каждой итерации выполняется `i = i - 1`

* Ограничения:
     * Переменная цикла должна быть объявлена до цикла.
     * Типы начального значения/границы и переменной цикла должны быть целыми числами (`int`).

**`break` и `continue`:**

* Допускаются только внутри тела циклов.
* `break` завершает ближайший внешний цикл.
* `continue` переходит к следующей итерации ближайшего внешнего цикла.

```dea+e
int i = 1;

while (i <= 5)
{
    print(i);
    i = i + 1;
}
```

```dea+e
int i;
for (i = 1 to 9)
{
    print(i);
}
```

```dea+e
int i;
for (i = 9 downto 1)
{
    print(i);
}
```

## 11. Функции и процедуры

**Объявление до использования:**

* Функция/процедура должна быть объявлена до её использования.

**Рекурсия:**

* Запрещены рекурсивные вызовы: функция не может вызывать саму себя.
* Запрещена взаимная рекурсия: функции не могут вызывать друг друга циклически.

**`return`:**

* Функции с возвращаемым значением должны завершаться инструкцией `return` с выражением совместимого типа.
* В процедурах допускается `return` без выражения (или отсутствие `return`, если управление достигает конца тела).

```dea+e
func int add(int a, int b)
{
    return a + b;
}

proc printResult(int value)
{
    print("сумма = ", value);
}

func int main()
{
int sum = add(x, y);
printResult(sum);

return 0;
}
```

## 12. Параметры и аргументы функций/процедур

* Функции и процедуры могут иметь параметры или быть без них.
* Объявление параметра содержит указание типа данных: `type name`. 
* Количество аргументов вызова должно соответствовать количеству параметров.
* Аргументы функции вычисляются слева направо.
* Порядок вычисления аргументов важен при наличии побочных эффектов — например, из-за вызова ещё одной функции: `sum(input(a), input(b))`.
